Welche Integritätsbedingungen können beim Anlegen der Tabelle
Stundenplan (Tag, Von, Bis, Raum, Dozent, Modul, ...)
mit CREATE TABLE definiert werden?
Hinweis: Unter Verwendung von NOT NULL, UNIQUE, PRIMARY / FOREIGN KEY und Datentypen.

  1. Ein Dozent darf nicht gleichzeitig in zwei Räumen eine Veranstaltung haben.
  2. Ein Dozent darf nicht gleichzeitig in zwei Räumen eine Veranstaltung haben, außer in Laboren.
  3. Für jede Veranstaltung muss es einen Raum und einen Dozenten geben.
  4. Für jedes Modul muss es mindestens zwei Veranstaltungen geben.
  5. Bei der Änderung eines Raums für eine Veranstaltung (UPDATE) muss der neue Raum den selben Typ wieder alte Raum haben (z.B. Hörsaal).
